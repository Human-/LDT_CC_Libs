Module fs

FileSystem API

The fs API allows access to the ComputerCraft FileSystem.

Notes:

  1. All of these functions except for fs.combine refer solely to absolute paths. This means that the current working directory, as set by the cd command or the shell.setDir function, is ignored. Each path name consists of a list of nonempty path components separated by forward slashes, and those path components are taken one by one with the first being contained in the root directory of the computer.
  2. If you need to deal with paths provided by the user that may be absolute or may be relative to the current working directory, use shell.resolve.
  3. Unlike most real-world operating systems, ComputerCraft's absolute path name system does not need to be started with a forward slash ( / ). Making the directory "a/b/c" is the same as making "/a/b/c". Leaving the slashes is just a matter of preference to the coder.

Type fs

Field(s)

fs.combine(basePath, localPath)

Combines two paths.

The new path consists of all the components of localPath inside all the components of basePath. Neither path needs to exist, this function only manipulates strings and does not touch the filesystem.

Parameters

  • #string basePath : base path.

  • #string localPath : path that will be appended to the base path.

Return value

#string: the combined path.

fs.copy(fromPath, toPath)

Copy a file or directory from one location to another.

The parent of the new location must be a directory, toPath must include the target filename and cannot be only a directory to move the file into, and toPath itself must not already exist.

Parameters

  • #path fromPath : location of the file or directory.

  • #path toPath : path the file or directory should be copied to.

fs.delete(path)

Deletes a file or directory.

A nonexistent target is ignored, a directory will be deleted along with all its contents.

Parameter

  • #path path : path of the directory that should be deleted.

fs.exists(path)

Checks if a path refers to an existing file or directory.

Parameter

  • #path path : The path of the file or directory that should be checked.

Return value

#boolean: true if the file exists, or false if not.

fs.getDrive(path)

Gets the type of storage medium holding a file or directory, or nil if the path does not exist.

Parameter

  • #path path : The path of the file or directory that should be checked.

Return value

#string: The type of storage medium holding the path, as of ComputerCraft 1.3 either "rom" or "hdd".

fs.getFreeSpace(path)

Gets the amount of space available on the path.

Parameter

  • #path path : The directory path that should be checked.

Return value

#number: The amount of space available on the path in bytes.

fs.getName(path)

Returns the last path component (forward-slash-separated) component of a path.

(the path doesn't need to exist).

Parameter

  • #path path : The path that should be parsed.

Return value

#path: The last path component of the path, or the string "root" if the path refers to the root directory.

fs.getSize(path)

Gets the size of a file.

Parameter

  • #path path : The path of the file that should be checked.

Return value

#number: The size of the file in bytes.

fs.isDir(path)

Checks if a path refers to an existing directory.

Parameter

  • #path path : The path of the file or directory that should be checked.

Return value

#boolean: true if it is a directory, or false if not.

fs.isReadOnly(path)

Checks if a path is read-only (i.e.

cannot be modified).

Parameter

  • #path path : The path of the file or directory that should be checked.

Return value

#boolean: false if path can be written to, or true if not.

fs.list(path)

Lists the directories and files in the given directory.

Parameter

  • #path path : path of the directory that should be listed.

Return value

#table: table of files and folders in path, which must be a directory.

fs.makeDir(path)

Creates a directory.

It fails if the location already exists as a file, it silently does nothing if the location or a parent already exists as a directory and it recursively creates directories if a parent is also missing.

Parameter

  • #path path : The path of the directory that should be created.

fs.move(fromPath, toPath)

Moves a file or directory to a new location.

The parent of the new location must be a directory, toPath must include the target filename and cannot be only a directory to move the file into, and toPath itself must not already exist.

Parameters

  • #path fromPath : The location of the file or directory.

  • #path toPath : The path the file or directory should be moved to.

fs.open(path, mode)

This function opens a file, in the mode specified in the string mode.

It returns a new file handle, or, in case of errors, nil plus an error string. The mode string can be any of the following:

  • "r": read mode (the default);
  • "w": write mode;
  • "a": append mode;

The mode string can also have a "b" at the end, which will open the file in binary mode.

Parameters

  • #path path : path to the file.

  • #string mode : used to specify the open mode.

Return values

  1. #file: a file handle.

  2. #nil: nil if the mode was read-only and the file did not exist.

file.read()

Reads the remaining text or a single byte from the file.

Notes:

  1. Only available in mode "rb"

Return values

  1. #number: the byte read from the file.

  2. #nil: nil if there are no more bytes in the file.

file.readAll()

Reads the all the text in the file.

Note: only available in mode "r"

Return values

  1. #string: the entire rest of the file, with the end-of-line character on the very last line (if present) stripped.

  2. #nil: nil if there are no more lines in the file.

file.readLine()

Reads the next line from the file.

Note: only available in mode "r"

Return values

  1. #string: the next line read from the file, with the end-of-line character stripped.

  2. #nil: nil if there are no more lines in the file.

file.write(data)

Writes a single byte or a string of characters exactly as they appear in the string data to the file.

Notes:

Parameter

file.writeLine(data)

Writes a string of characters to the file, then appends an end-of-line character.

Notes:

  1. Only available in mode "w" or "a"

Parameter
