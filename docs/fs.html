Module fs

The FileSystem API allows you to mess around with the, you guessed it, file system.
All of these functions except for fs.combine refer solely to absolute paths. This means that the current working directory, as set by the cd command or the shell.setDir function, is ignored. Each path name consists of a list of nonempty path components separated by forward slashes, and those path components are taken one by one with the first being contained in the root directory of the computer. If you need to deal with paths provided by the user that may be absolute or may be relative to the current working directory, use shell.resolve. Unlike most real-world operating systems, ComputerCraft's absolute path name system does not need to be started with a forward slash ( / ). Making the directory "a/b/c" is the same as making "/a/b/c". Leaving the slashes is just a matter of preference to the coder.

Type fs

Field(s)

fs.combine(basePath, localPath)
Combines two paths.
The new path consists of all the components of localPath inside all the components of basePath. Neither path needs to exist, this function only manipulates strings and does not touch the filesystem.

Parameters

  • #string basePath : base path.
  • #string localPath : path that will be appended to the base path.

Return value

#string: the combined path.

fs.copy(fromPath, toPath)
Copy a file or directory from one location to another.
The parent of the new location must be a directory, toPath must include the target filename and cannot be only a directory to move the file into, and toPath itself must not already exist.

Parameters

  • #string fromPath : location of the file or directory.
  • #string toPath : path the file or directory should be copied to.
fs.delete(path)
Deletes a file or directory.
A nonexistent target is ignored, a directory will be deleted along with all its contents.

Parameter

  • #string path : path of the directory that should be deleted.
fs.exists(path)
Checks if a path refers to an existing file or directory.

Parameter

  • #string path : The path of the file or directory that should be checked.

Return value

#boolean: True if the file exists, or false if not.

fs.getDrive(path)
Gets the type of storage medium holding a file or directory, or nil if the path does not exist.

Parameter

  • #string path : The path of the file or directory that should be checked.

Return value

#string: The type of storage medium holding the path, as of ComputerCraft 1.3 either "rom" or "hdd".

fs.getName(path)
Returns the last path component (forward-slash-separated) component of a path (the path doesn't need to exist).

Parameter

  • #string path : The path that should be parsed.

Return value

#string: The last path component of the path, or the string "root" if the path refers to the root directory.

fs.getSize(path)

Checks if a path refers to an existing directory.

cannot be modified).

fs.list(path)
Lists the directories and files in the given directory.

Parameter

  • #string path : Path of the directory that should be listed.

fs.makeDir(path)
Creates a directory.
It fails if the location already exists as a file, it silently does nothing if the location or a parent already exists as a directory and it recursively creates directories if a parent is also missing.

Parameter

  • #string path : The path of the directory that should be created.
fs.move(fromPath, toPath)
Moves a file or directory to a new location.
The parent of the new location must be a directory, toPath must include the target filename and cannot be only a directory to move the file into, and toPath itself must not already exist.

Parameters

  • #string fromPath : The location of the file or directory.
  • #string toPath : The path the file or directory should be moved to.
fs.open(path, mode)
This function opens a file, in the mode specified in the string mode.
It returns a new file handle, or, in case of errors, nil plus an error message. The mode string can be any of the following:

  • "r": read mode (the default);
  • "w": write mode;
  • "a": append mode;

The mode string can also have a "b" at the end, which will open the file in binary mode.

Parameters

  • #string path : path to the file.
  • #string mode : used to specify the open mode.

Return value

#file: a file handle.
