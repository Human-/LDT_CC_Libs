Module peripheral

Peripheral API

The peripheral API is for interacting with external peripherals, such as the Disk Drive, the Advanced Monitor and Monitor.

Notes:

  1. Computers and Turtles are also considered peripherals and you can interface two adjacent Computers, although in a more limited fashion compared to rednet.

Type peripheral

peripheral.getMethods(side)

Returns a table with all available methods for the peripheral on side.

Tells you what is attached to a side of a computer.

Returns a handle to a peripheral that can then be used to call methods, as if using peripheral.call(side, method)

Type peripheral

Field(s)

peripheral.call(side, methodName, ...)

Calls a method on the peripheral attached to side.

Parameters

  • #side side : the side of the computer to consider.

  • #string methodName : the method to be called on the peripheral located on side.

  • ... : arguments to be supplied to methodName.

Return value

... : Return values match those of called method.

peripheral.getMethods(side)

Returns a table with all available methods for the peripheral on side.

Parameter

  • #side side : the side of the computer to consider.

Return value

#table: Returns a table containing all methods for peripheral on side.

peripheral.getType(side)

Tells you what is attached to a side of a computer.

Parameter

  • #side side : the side of the computer to consider.

Return values

  1. #string: Returns the type or peripheral present on side ("modem", "computer", "turtle", "drive", "monitor", "printer", "command").

  2. #nil: Returns nil if side is empty.

peripheral.isPresent(side)

Tells you whether a peripheral is present on a side of the computer or the turtle.

Parameter

  • #side side : the side of the computer to consider.

Return value

#boolean: Returns true if a peripheral is present on side.

peripheral.wrap(side)

Returns a handle to a peripheral that can then be used to call methods, as if using peripheral.call(side, method)

Parameter

  • #side side : the side of the computer to consider.

Return values

  1. #computer: Returns a handle to the computer peripheral located on side.

  2. #advcomputer: Returns a handle to the advcomputer peripheral located on side.

  3. #monitor: Returns a handle to the monitor peripheral located on side.

  4. #advmonitor: Returns a handle to the advmonitor peripheral located on side.

  5. #printer: Returns a handle to the printer peripheral located on side.

  6. #modem: Returns a handle to the modem peripheral located on side.

  7. #command: Returns a handle to the command peripheral located on side.

modem.open(side)

Tells the Computer that the side can be used for networking.

Parameter

  • #side side : the side of the computer to consider.

modem.receive(timeoutSeconds)

Waits until it either receives a rednet message or the timeout has passed.

Notes:

  1. Leave args empty to wait for a message forever.
  2. senderID and message are set to nil if a timeout occurs.
  3. To get the computerID from a computer see os#computerID or the id program.
  4. You must be calling this function at the moment a message is sent, or you will miss it. If you need to watch for messages while doing something else at the same time, see parallel#waitForAny.

Parameter

  • os#time timeoutSeconds : Seconds to wait for a message. If timeoutSeconds is nil, waits forever.

Return value

os#computerID, #string, #number: senderID, message, distance-to-sender (or nil if using redpower bundled cable instead of a modem)

modem.send(receiverID, message, WaitUntilPortOpen)

Sends a string to a specific Computer or Turtle using the opened sides.

Notes:

  1. To get the computerID from a Computer see os#computerID or the id program.

Parameters

  • os#computerID receiverID : computerID of the Computer or Turtle to be communicated with.

  • #string message : string to be sent to the Computer or Turtle matched by receiverID.

  • #boolean WaitUntilPortOpen : Setting WaitUntilPortOpen to true can help when the server is not receiving a response from the responding client.

Return value

#boolean: true if a string has been sent, false if not. true does not guarantee that the destination was reachable!
