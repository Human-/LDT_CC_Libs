Module io

IO API

The io API is a reimplementation of the default Lua input / output API with some features trimmed.

Type io

io.open(path, mode)

Opens a file path for reading/writing.
io.read(format)

Reads input from the user.
io.type_(handle)

Returns whether the given file is opened, closed, or invalid.
io.write(text)

Writes a string to the terminal.

Type io

Field(s)

io.open(path, mode)

Opens a file path for reading/writing.

Valid Modes:

  1. r - read only
  2. w - write only
  3. a - equivalent to w
  4. rb - binary read only
  5. wb - binary write only
  6. ab - equivalent to wb

Parameters

  • fs#path path : path` of the file to be opened.

  • #string mode : string mode of the opened file. r,w,a,rb,wb, or ab supported.

Return value

fs#file: file handle for reading/writing the opened file.

io.read(format)

Reads input from the user.

Notes:

  1. This is simply a very thin wrapper around read.

Parameter

  • #string format : ignored parameter, always uses "*l".

Return value

#string: entire input in a string.

io.type_(handle)

Returns whether the given file is opened, closed, or invalid.

Notes:

  1. The function is actually named type, not type_. type_ is used due to a limitation of the LDT documentation language. Remove the trailing underscore for the function to work!

Parameter

  • fs#file handle : file to be queried.

Return values

  1. #string: "closed file" if file is closed, otherwise "file".

  2. #nil: nil if the handle is invalid.

io.write(text)

Writes a string to the terminal.

Notes:

  1. This is simply a very thin wrapper around write.

Parameter

  • #string text : string to be written to the terminal.
