Module redstone

Redstone API

The redstone API contains methods to control attached RedPower cables/bundled cables and regular redstone.

Notes:

  1. The rs module is an alias of redstone.

Type redstone

redstone.getBundledInput(side)

Returns the state of a redpower wire inside a bundle connected to side.
redstone.getBundledOutput(side)

Returns the set of redpower wires inside a bundle on side that are being driven high by the local console.
redstone.getInput(side)

Returns the current redstone input signal state on side.
redstone.getOutput(side)

Returns the current redstone output signal on side.
redstone.getSides()

Returns an array of possible sides.
redstone.setBundledOutput(side, colors)

Powers one or multiple colored signals in a redpower bundled wire connected to side.
redstone.setOutput(side, state)

Set or reset a redstone signal on side.
redstone.testBundledInput(side, color)

Returns true or false whether or not a colored signal is active in a redpower bundled wire connected to side.

Type redstone

Field(s)

redstone.getBundledInput(side)

Returns the state of a redpower wire inside a bundle connected to side.

Parameter

  • peripheral#side side : the side of the computer to consider.

Return value

colors#color: The colors/colours that the computer is getting redstone power from on the given side.

redstone.getBundledOutput(side)

Returns the set of redpower wires inside a bundle on side that are being driven high by the local console.

Notes:

  1. Not those that are driven high by another device on the bundle but
 not driven high by the local console.

Parameter

  • peripheral#side side : the side of the computer to consider.

Return value

colors#color: The colors/colours that the computer is sending redstone power to on the given side.

redstone.getInput(side)

Returns the current redstone input signal state on side.

Notes:

  1. This function will throw an error if you give it an invalid side.

Parameter

  • peripheral#side side : the side of the computer to consider.

Return value

#boolean: true if the computer is getting redstone power from the given side; otherwise false.

redstone.getOutput(side)

Returns the current redstone output signal on side.

Parameter

  • peripheral#side side : the side of the computer to consider.

Return value

#boolean: true if the computer is sending redstone power from the given side; otherwise false.

redstone.getSides()

Returns an array of possible sides.

Return value

#table: list of all valid sides.

redstone.setBundledOutput(side, colors)

Powers one or multiple colored signals in a redpower bundled wire connected to side.

Notes:

  1. In order to power multiple signals, add the color/colour values of the colors/colours you want to activate.
  2. To turn off a signal, subtract that color/colour from getBundledOutput's return (if present) and use that number for the integer.
  3. To turn off all of the values, use 0 for the integer.

Parameters

  • peripheral#side side : the side of the computer to consider.

  • colors#color colors : The colors/colours to power.

redstone.setOutput(side, state)

Set or reset a redstone signal on side.

Notes:

  1. This function will throw an error if you give it an invalid side.

Parameters

  • peripheral#side side : the side of the computer to consider.

  • #boolean state : true powers the side; false depowers the `side.

redstone.testBundledInput(side, color)

Returns true or false whether or not a colored signal is active in a redpower bundled wire connected to side.

Parameters

  • peripheral#side side : the side of the computer to consider.

  • colors#color color : The color/colour to check.

Return value

#boolean: true if the computer is getting redstone power from the given side from the provided color/colour; otherwise false.
